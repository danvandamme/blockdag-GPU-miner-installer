DagTech GPU Miner — Windows Installer

Installs the DagTech GPU Miner on Windows. Supports NVIDIA, AMD, and Intel GPUs via OpenCL. CPU mining runs in parallel. A local control server and dashboard are included.

Quick Start

  1. Right-click install-gpu-miner.bat → Run as administrator
  2. Enter your wallet address, pool host, and worker name when prompted
  3. Choose a start mode (see below)
  4. Open the dashboard at http://127.0.0.1:8883 while mining

Dashboard

Open http://127.0.0.1:8883 in any browser while the miner is running.

  • Live CPU and GPU hashrate (shown separately)
  • Share counts (submitted / accepted / rejected / stale)
  • Pool connection status and current difficulty
  • Config editor — change settings without editing files manually
  • Update button — downloads and applies the latest version from GitHub
  • Live log viewer

If the dashboard is blank after an update, use Restart Control and refresh.

Updating

Open the dashboard at http://127.0.0.1:8883, click Update, and confirm. The control server downloads the latest version, applies it, and restarts. The miner keeps running during the update.

If the dashboard is unreachable after a failed update, replace the control script manually in an elevated PowerShell window: 
$url = "https://raw.githubusercontent.com/danvandamme/blockdag-GPU-miner-installer/main/dagtech-control.ps1"
Invoke-WebRequest $url -OutFile "C:\dagtech-gpu-miner\bin\dagtech-control.ps1" -UseBasicParsing
Restart-ScheduledTask "DagTech GPU Miner"

Troubleshooting

GPU shows 0 H/s

Auto-detected. The miner checks GPU hashrate 90 seconds after starting. If GPU is 0 H/s but CPU is running, it automatically writes GPU_PLATFORM=1 to config.env and restarts. Watch the dashboard log for a “Watchdog: GPU hashrate is 0 H/s … switching to GPU_PLATFORM=1” message.

If GPU is still 0 H/s after a second start, open config.env and try GPU_PLATFORM=2. On AMD systems, Intel integrated graphics is often platform 0 and the Radeon card is platform 1 or 2.

No OpenCL platforms found

Windows maintains a registry list at HKEY_LOCAL_MACHINE\SOFTWARE\Khronos\OpenCL\Vendors that tells the OpenCL loader which GPU driver DLLs to use. If this list is empty (common after driver updates or clean Windows installs), the miner cannot find any GPU and reports 0 H/s or an OpenCL error.

The installer fixes this automatically. If the problem persists, re-run install-gpu-miner.bat as Administrator. If you need to fix it without reinstalling, run the appropriate script below in an elevated PowerShell window.

NVIDIA

$dll = Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository" `
    -Recurse -Filter "nvopencl64.dll" -ErrorAction SilentlyContinue |
    Select-Object -First 1 -ExpandProperty FullName
if ($dll) {
    New-Item -Path "HKLM:\SOFTWARE\Khronos\OpenCL\Vendors" -Force | Out-Null
    New-ItemProperty -Path "HKLM:\SOFTWARE\Khronos\OpenCL\Vendors" `
        -Name $dll -Value 0 -PropertyType DWORD -Force
    Write-Host "Done: $dll"
} else {
    Write-Host "nvopencl64.dll not found - reinstall your NVIDIA drivers."
}

AMD

$dll = Get-ChildItem "C:\Windows\System32" -Filter "amdocl64.dll" -ErrorAction SilentlyContinue |
    Select-Object -First 1 -ExpandProperty FullName
if (-not $dll) {
    $dll = Get-ChildItem "C:\Windows\System32\DriverStore\FileRepository" `
        -Recurse -Filter "amdocl64.dll" -ErrorAction SilentlyContinue |
        Select-Object -First 1 -ExpandProperty FullName
}
if ($dll) {
    New-Item -Path "HKLM:\SOFTWARE\Khronos\OpenCL\Vendors" -Force | Out-Null
    New-ItemProperty -Path "HKLM:\SOFTWARE\Khronos\OpenCL\Vendors" `
        -Name $dll -Value 0 -PropertyType DWORD -Force
    Write-Host "Done: $dll"
} else {
    Write-Host "amdocl64.dll not found - reinstall AMD Radeon Software from amd.com/support"
}

GPU intensity — crashes or out of memory

If the miner crashes or the GPU shows errors, GPU_INTENSITY may be too high for your VRAM. Lower it in steps of 10 in config.env until stable. The installer sets a recommended value based on detected VRAM (~75% usage). If VRAM detection was inaccurate, start at 30 and increase gradually.

Miner not starting after reboot (service mode)

  1. Open Task Scheduler (search in Start Menu)
  2. Find DagTech GPU Miner in the task list
  3. Check the status is Ready and last run result is 0x0
  4. If the task is missing, re-run the installer and choose Service mode

Dashboard not loading

  1. Check that the miner is running — look for powershell.exe and dagtech-gpu-miner.exe in Task Manager
  2. Use the Restart Control desktop shortcut and try http://127.0.0.1:8883 again
  3. Check C:\dagtech-gpu-miner\logs\ for errors

Antivirus blocking the miner

Mining software is frequently flagged as a false positive. Add an exclusion rather than allowing individual detections — exclusions persist across updates.

Windows Defender — handled automatically by the installer. If the binary is removed anyway:

  1. Open Windows Security → Virus & threat protection → Protection history
  2. Find the quarantined item and select Allow
  3. Re-run the installer to restore the binary

To add Defender exclusions manually (elevated PowerShell):

Add-MpPreference -ExclusionPath "C:\dagtech-gpu-miner"
Add-MpPreference -ExclusionProcess "C:\dagtech-gpu-miner\bin\dagtech-gpu-miner.exe"

ESET and other third-party AV — add C:\dagtech-gpu-miner as a folder exclusion in your AV settings. Restore any quarantined files before re-running the installer.

If the miner binary is quarantined, the watchdog auto-restart will fail silently because the .exe is missing. Always check the AV quarantine first if the miner stops and does not restart on its own.

Uninstall

Double-click the DagTech GPU Miner - Uninstall desktop shortcut, or run:

C:\dagtech-gpu-miner\bin\dagtech-uninstall.bat

This stops the miner, removes the scheduled task, deletes all installed files, and removes the desktop shortcuts.
